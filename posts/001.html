¶10 内联函数

¶10.1 inline

在 Kotlin 中，inline 是一个关键字，用于修饰函数。
使用 inline 修饰的函数在编译时会进行内联优化。这意味着函数的代码会被直接插入到调用它的地方，而不是通过常规的函数调用机制。这样可以减少函数调用的开销，提高程序的性能。
内联函数适用于那些短小且被频繁调用的函数。但需要注意的是，过度使用内联可能会导致生成的代码体积增大。

inline的前世今生

先来回顾一下：什么是高阶函数？

高阶函数：高阶函数是指函数的参数是一个函数或者返回值是一个函数

查看android中的一些高阶函数，如apply等，会发现它们身上有一个inLine，这是为什么呢？
因为要做性能优化，为什么要给高阶函数做性能优化呢？

这就要从kotlin实现高阶函数的原理来说起了，Java不支持高阶函数，kotlin是如何实现的呢？

kotlin其实是创建了一个匿名内部类对象，用这个对象去执行lambda表达式的内容，这就产生了一个性能问题，每调用一次高阶函数，就会产生一个匿名内部类对象，如何解决这个性能问题呢？于是就产生了inline

当我们将inline标记在一个高阶函数上的时候，编译器就会做内联优化，直接平铺代码，不会产生多余的匿名内部类对象

inline内联优化示意图

总结：如果我们自己编写了一个高阶函数，一般可以加上inline实现内联优化，所付出的代价就是编译的字节码会增大包体积大小

¶10.2 noinline

noinline : 关闭 inline ，即关闭内联优化。那么，究竟在何种情况下需要关闭内联优化呢？
最为常见的情形是：在针对内联高阶函数的操作中，若将函数参数中的 lambda 表达式当作返回值，这是不被许可的。原因在于，执行 lambda 表达式的匿名对象已因 “内联” 而被消除，根本无法找到相应对象去执行此 lambda 表达式，所以禁止将其作为返回值。而当关闭内联优化之后，便会依照匿名内部类对象的逻辑来执行 lambda 表达式。

一般当我们在一个内联函数里进行了一些风骚操作，编译报错的时候，试着加上noinline试试

通常而言，当我们于一个内联函数中实施了某些独特而复杂的操作，进而导致编译出现错误之时，可以尝试添加 noinline 来解决问题。
“illegal usage”：意为非法使用。
例如在此处，将函数参数 block 作为对象进行返回，然而 block 会被内联展开，此时它已不再是一个对象了。所以在这种情况下，就需要添加 noinline ，关闭针对 block 的内联优化，使其能够以对象的形式返回。

¶10.3 crossinline

在 Kotlin 中，crossinline 关键字主要用于限制内联函数中传递的 lambda 表达式的行为。
当一个内联函数接收一个 lambda 表达式作为参数时，如果这个内联函数打算将这个 lambda 表达式传递给另一个函数，那么这个 lambda 表达式参数就需要使用 crossinline 进行修饰。
使用 crossinline 的主要目的是防止在这个 lambda 表达式内部使用非局部返回，也就是不允许通过 return 语句从包含这个 lambda 表达式的函数直接返回。
例如，如果一个内联函数 inlineFunction 接受一个 crossinline 的 lambda 表达式 action ，并将其传递给另一个函数 anotherFunction ，那么在 action 内部就不能使用 return 来提前结束 inlineFunction 的执行。
crossinline 有助于确保代码的逻辑和控制流更加清晰和可预测，避免意外的控制流跳转，特别是在涉及内联函数和 lambda 表达式的复杂代码结构中。

  • crossinline ：由于函数参数被间接调用时，return 作用域会引发迷惑性，所以 Kotlin 禁止函数参数间接调用，使用 crossline 可以突破这个限制，但是函数体不可以再使用 return

¶11.3 in和out

out关键字
out 关键字用于表示一个类型参数只能作为输出使用。这意味着你可以从该类型的实例中返回类型参数，但不能将类型参数作为输入参数

什么时候使用 out？
当你有一个类型，只需要从中获取值，而不需要向其中传递值时，可以使用 out

在这个例子中，Producer<out T> 表示你可以从 Producer 中获取 T 类型的对象，但不能将其作为输入。

in关键字
in 关键字用于表示一个类型参数只能作为输入使用。这意味着你可以将类型参数传递给该类型的实例，但不能从中获取类型参数

什么时候使用 in？
当你有一个类型，只需要向其中传递值，而不需要从中获取值时，可以使用 in。

out 和 in 的区别

  • 输出（out）：

    • 允许从类型中获取值。
    • 适用于返回值的场景。
    • 例如，Producer<out T>。

  • 输入（in）：

    • 允许向类型中传递值。
    • 适用于参数的场景。
    • 例如，Consumer<in T>。

总结

  • 使用 out 时，你可以将类型参数作为返回类型，允许协变。
  • 使用 in 时，你可以将类型参数作为函数参数，允许逆变。

¶12 委托

Kotlin 中的委托（Delegation）。这是一个非常强大且有趣的特性，它可以让你的代码更加简洁和可维护。委托是一种设计模式，它允许一个对象将某些功能委托给另一个对象。简单来说，就是把某些任务交给其他对象来处理，而不是在本类中实现。这种方式可以减少代码重复，提高代码的复用性。

在 Kotlin 中，委托有两种主要形式：

  1. 属性委托（Property Delegation）
  2. 类委托（Class Delegation）

¶12.1 属性委托

属性委托是 Kotlin 的一个特性，它允许你将属性的 getter 和 setter 的实现委托给另一个对象。最常见的Kotlin内置委托是 lazy委托。lazy 委托用于延迟初始化属性。这意味着该属性在第一次被访问时才会被初始化。

在这个例子中，str 只在第一次访问时计算，后续访问将直接返回结果。
